# Projects

Real-world mini-projects that combine multiple AI skills.

## Why Projects?

Isolated exercises build skills. Projects build expertise.

Projects force you to:
- Chain multiple techniques together
- Handle real-world messiness
- Learn what actually matters
- Build something tangible

## Project Catalog

### ðŸŸ¢ Starter Projects

#### Project 1: Document a Function
**Goal:** Take an undocumented function and create complete documentation.

**Process:**
1. Ask AI to explain what the function does
2. Generate a docstring with parameters and return values
3. Create usage examples
4. Write a brief README section

**Skills practiced:**
- Code comprehension prompting
- Documentation generation
- Iterative refinement

**Deliverable:** Fully documented function ready for your codebase

---

#### Project 2: Debug Challenge
**Goal:** Fix a bug using only AI-assisted debugging.

**Process:**
1. Find a bug in your code (or create one intentionally)
2. Describe the symptoms to AI
3. Follow AI's diagnostic suggestions
4. Implement and verify the fix
5. Document what you learned

**Skills practiced:**
- Debugging workflows
- Context provision
- Verification habits

**Deliverable:** Fixed code + documented debugging process

---

#### Project 3: Explain Like I'm Five
**Goal:** Create a beginner-friendly explanation of a complex topic.

**Process:**
1. Pick a technical concept you understand well
2. Ask AI to explain it for beginners
3. Iterate to remove jargon
4. Add analogies and examples
5. Have a non-expert review it

**Skills practiced:**
- Audience targeting
- Iterative refinement
- Output quality assessment

**Deliverable:** Clear explanation you could share with beginners

---

### ðŸŸ¡ Intermediate Projects

#### Project 4: Code Review Workflow
**Goal:** Build a repeatable code review process.

**Process:**
1. Create a code review prompt template
2. Test on 3 different code samples
3. Refine based on what you learn
4. Add to your prompt library
5. Document the workflow

**Skills practiced:**
- Template creation
- A/B testing prompts
- Workflow design
- Library building

**Deliverable:** Reusable code review workflow + prompt template

---

#### Project 5: Research & Write
**Goal:** Research a topic and write a blog post with AI assistance.

**Process:**
1. Use AI to explore the topic space
2. Create an outline
3. Research key points (verify claims!)
4. Draft each section with AI
5. Edit and add your voice
6. Final polish

**Skills practiced:**
- Research synthesis
- Outline-first writing
- Fact verification
- Multi-step workflows

**Deliverable:** Publishable blog post (1000-1500 words)

---

#### Project 6: Refactor Legacy Code
**Goal:** Refactor messy code into clean, maintainable code.

**Process:**
1. Have AI analyze the existing code
2. Identify code smells and issues
3. Plan the refactoring steps
4. Refactor incrementally with AI help
5. Verify behavior is preserved
6. Document the improvements

**Skills practiced:**
- Code analysis
- Planning with AI
- Incremental changes
- Verification

**Deliverable:** Refactored code + before/after comparison

---

#### Project 7: Build a CLI Tool
**Goal:** Create a small command-line tool from scratch.

**Process:**
1. Define what the tool should do
2. Design the interface with AI
3. Implement core functionality
4. Add error handling
5. Write usage documentation
6. Test edge cases

**Skills practiced:**
- Code generation
- Iterative development
- Documentation
- Testing with AI

**Deliverable:** Working CLI tool you can actually use

#### Project 8: Multi-Model Pipeline
**Goal:** Build a workflow that uses multiple AI models strategically.

**Process:**
1. Identify a complex task
2. Break into subtasks

- Task decomposition

#### Project 9: Create a Course Module
**Goal:** Design a learning module for teaching something.

**Process:**
1. Define learning objectives
2. Create outline with AI
3. Develop content for each section
4. Create exercises and quizzes
5. Build assessment rubrics
6. Test with a learner

**Skills practiced:**

- Structured output
- Multi-part workflows
- Quality assessment

**Deliverable:** Complete learning module ready to teach

**Goal:** Migrate code from one framework/language to another.

**Process:**

2. Plan migration strategy
3. Create conversion templates
4. Migrate incrementally
5. Verify functionality
6. Document gotchas

**Skills practiced:**
- Large context management
- Code transformation
- Verification workflows
- Edge case handling

**Deliverable:** Migrated codebase + migration guide
